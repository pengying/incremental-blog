## Project Overview

A personal blog platform built with Next.js 13 (Pages Router), Material-UI, and MDX. Content lives entirely in `.mdx` files—there is no database. The site is statically generated at build time and deployed to Cloudflare Pages.

- **Live site:** https://incremental.fyi
- **Author:** Peng Ying (configured in `incremental.config.js`)
- **Status:** Active personal blog (WIP per README)

## Content: Writing Blog Posts

### File Location

Posts are MDX files placed at:
```
posts/YYYY/M/your-post-filename.mdx
```

### Required Frontmatter

Every post needs YAML frontmatter:

```yaml
---
title: "Your Post Title"
date: "2024-01-15"        # ISO date string (YYYY-MM-DD)
author: "Peng Ying"
hero: "/images/hero.jpg"  # Path relative to /public
publish: true             # Set false to hide from the site
excerpt: "Short description shown in post listing."
tags: ["tag1", "tag2"]
---
```

- `publish: false` posts are loaded but not shown in listings
- `slug` is optional; if omitted it's auto-generated by `slugify(title.toLowerCase())`
- Dates must be strings in frontmatter (not YAML date objects) due to a known limitation in `lib/posts.js`

### MDX Features Available

In post content you can use:
- **GitHub Flavored Markdown** (tables, strikethrough, task lists)
- **Emoji** via `:emoji_name:` syntax
- **Syntax-highlighted code blocks** via Prism (VS Code dark theme)
- **Mermaid diagrams** via `<div class="mermaid">` blocks (loaded from CDN)
- **Image sizing classes**: `Image__Small`, `Image__Medium`, `Image__Large`, `Image__XSmall`, `Image__With-Shadow`

### Custom MDX Components

The following MDX element overrides are active in `pages/posts/[slug].tsx`:

| MDX element | Rendered by |
|---|---|
| `img` / `<Image>` | `ResponsiveImage` (Next.js Image, full-width) |
| `a` | MUI `Link` wrapping Next.js `Link` |
| `p` | `components/paragraph.tsx` |
| `h1`–`h6` | `components/headings.tsx` (h1 renders as h2 style) |
| `blockquote` | `components/blockquote.tsx` |
| `hr` | `components/horizontalrule.tsx` |
| `ol` / `ul` | `components/lists.tsx` |
| `table` / `thead` / `th` / `td` | `components/table.tsx` |

## Architecture: How Posts Are Loaded

All post loading is in `lib/posts.js`. Key functions:

- **`getSortedPostsHeaders(filter?)`** — Recursively reads all `.md`/`.mdx` files under `/posts`, parses frontmatter with `gray-matter`, calculates reading time, normalizes dates, generates slugs. Results are **module-level cached** (runs once per server/build process). Pass `filter=true` to return only `publish: true` posts.
- **`getPostData(slug)`** — Returns a single post's data by slug.
- **`getPostSlugs()`** — Returns `{ params: { slug } }` array for Next.js `getStaticPaths`.
- **`getCopyrightDate()`** — Derives the copyright year range from earliest/latest post dates.

MDX serialization happens in `pages/posts/[slug].tsx` via `next-mdx-remote/serialize` with the remark/rehype plugin chain.

## Styling Conventions

### Theme

The site uses a **dark theme** defined in `styles/theme/mui-theme.ts`:
- Background: `#111216`
- Card/surface: `#1D2128`
- Primary fonts: Roboto (UI), Merriweather (headings/quotes)

### Custom MUI Typography Variants

Extend `Typography` with these custom variants when building UI:
- `logo` — Site logo text
- `postTitle` — Large post heading
- `quote` — Pull quote style
- `heroName` — Author name in hero
- `authorName` — Inline author attribution

### Styling Approaches (in priority order)

1. **MUI `sx` prop** — Preferred for component-level responsive styles
2. **`styled()` from MUI** — For reusable styled components
3. **`styled-jsx`** — Used for global CSS within a component (e.g., Prism styles in post page)
4. **Tailwind** — Available but minimally used; prefer MUI for consistency

### TypeScript Path Aliases

Use `@/` to import from the project root:
```ts
import Layout from "@/components/layout";
import { getPostData } from "@/lib/posts";
```

## Key Conventions

### TypeScript

- Strict mode is enabled (`tsconfig.json`)
- Shared interfaces are in `types/index.d.ts` (`IAuthor`, `IArticle`)
- `any` types appear in existing code (e.g., `frontmatter: any`); avoid adding new ones

### Components

- All components are **functional components** (no class components)
- Components use MUI's `useTheme()` hook for theme access (import from `@emotion/react` in the post page, `@mui/material/styles` elsewhere)
- Export styled components as named constants at the bottom of the file

### Routing

- This is a **Pages Router** Next.js app (not App Router)
- All static data fetching uses `getStaticProps` / `getStaticPaths`
- No API routes exist — this is a purely static site

### Analytics & Third-Party

- Google Analytics GA4: ID `G-QKCWGT5WXP` hardcoded in `pages/_app.tsx`
- Disqus comments: configured but **commented out** in `pages/posts/[slug].tsx`
- Mermaid: loaded from CDN via `<Script>` in the post page
